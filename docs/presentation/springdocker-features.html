Config-first. Four commands.

Strategy in git. Output in CI.

configure

Interactive wizard writes [dockerfile] to .dockly.toml — reviewable in PRs.

$ dockly configure --project-root .

dockerfile generate

Non-interactive. Reads resolved config. Writes Dockerfile and .dockerignore.

$ dockly dockerfile generate --project-root .

explain

Advisory static analysis — recognized optimizations and config linkage for review, not a security audit.

$ dockly explain ./Dockerfile --config-aware

verify

Lint · scan · layer audit · signature · SBOM · smoke · config drift. One pass/fail.

$ dockly verify ./Dockerfile --check-config-drift

The quick experience

From git clone to a production image.

  1. Inspect. dockly inspect reads pom.xml / build.gradle[.kts] and reports compatibility signals.
  2. Initialize. dockly init writes a starter .dockly.toml (or use init --interactive to chain configure).
  3. Configure. dockly configure picks a profile and writes explicit [dockerfile] options — the SSOT your team reviews in PRs.
  4. Generate. dockly dockerfile generate reads config deterministically and writes the Dockerfile.
  5. Verify. dockly verify --check-config-drift runs built-in checks and confirms the Dockerfile matches config.
  6. Benchmark. dockly benchmark generate/run/analyze provides reproducible evidence for tuning decisions.

What you get

A Dockerfile you'd write — if you had three days and a security team.

# Generated by dockly — https://github.com/mnafshin/dockly
# Java 25 | build-tool: maven

FROM --platform=$BUILDPLATFORM eclipse-temurin:25-jdk@sha256:… AS build
WORKDIR /app
COPY mvnw pom.xml ./
COPY .mvn ./.mvn
RUN --mount=type=cache,sharing=locked,target=/root/.m2 ./mvnw -B -q package -DskipTests
RUN java -Djarmode=layertools -jar /app/target/*.jar extract --destination /layers

FROM --platform=$BUILDPLATFORM eclipse-temurin:25-jdk@sha256:… AS jre-builder
RUN jdeps … && jlink --add-modules "$MODULES" … --output /jre/out

FROM --platform=$TARGETPLATFORM gcr.io/distroless/base-debian13:nonroot
WORKDIR /app
COPY --from=build /layers/… ./
COPY --from=jre-builder /jre/out /opt/java
ENV JAVA_HOME=/opt/java PATH="${JAVA_HOME}/bin:${PATH}"
USER nonroot
STOPSIGNAL SIGTERM
ENTRYPOINT ["java", "-XX:MaxRAMPercentage=75", "-XX:+ExitOnOutOfMemoryError", \
  "-Djava.io.tmpdir=/tmp", "org.springframework.boot.loader.launch.JarLauncher"]

Abbreviated from the Java 25 sample recipe. Your service can use Java 17+ — set java_version in config. Probe readiness in Kubernetes — no shell HEALTHCHECK on distroless.
